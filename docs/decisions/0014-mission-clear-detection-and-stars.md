# 0014. Mission clear detection & star rating

- Status: Accepted
- Date: 2026-07-08
- Deciders: 차니, Claude

## Context

[ADR 0003](0003-mission-system-in-mvp.md)에서 미션을 MVP에 넣기로 했고, Phase 5는
그 게임 레이어를 완성하는 단계다. 세 미션(튜토리얼 / 3-tier / 서버리스)이 실제로
"완료"로 판정되고 별점이 매겨져야 한다. 판정 로직을 어디에 두고, 별점 기준을 어떻게
일관되게 잡을지 정해야 한다.

## Decision

판정을 **미션별 `check` 함수**로 데이터화한다. `Mission.check(ctx)`가 0–3 별점을
반환하고, 0 = 미완료, ≥1 = 클리어다. `MissionCheckContext`는 현재 그래프
(`nodes`/`edges`), 트래픽 시뮬레이션 결과(`sim`, ADR 0012 재사용), 전체 설정
유효성(`allValid`, ADR 0011의 `validate` 재사용)을 담는다. MissionPanel이 그래프가
바뀔 때마다 이 컨텍스트를 만들어 각 미션의 별점을 **실시간**으로 표시한다(완료
배지 + 채워진 별).

별점 루브릭(미션마다 목표는 다르지만 층위는 공통):

- **★1** — 목표 토폴로지 달성(튜토리얼: VPC 안 퍼블릭 Subnet / 3-tier: ALB→EC2→RDS
  트래픽 도달 / 서버리스: Lambda→저장소 도달).
- **★2** — 모든 노드가 설정 검증을 통과(`allValid`).
- **★3** — 미션별 "잘 만든" 보너스(튜토리얼: IGW로 인터넷 연결 / 3-tier: Security
  Group 배치 / 서버리스: S3에 저장).

## Consequences

- **좋은 점**: 미션 판정이 각 미션 파일에 데이터로 모여 있고, 시뮬레이션·검증 결과를
  재사용해 중복이 없다. 그래프를 편집하면 별점이 즉시 갱신되어 게임처럼 피드백된다.
  새 미션은 `check`만 추가하면 된다.
- **나쁜 점 / 한계**: 판정이 시뮬레이터의 그리디 단일 경로(ADR 0012)에 의존하므로,
  시뮬레이터가 보지 못하는 구성(분기·복수 경로)은 별점에 반영되지 않는다. 루브릭은
  MVP 데모용으로 단순하다.
- **후속 영향**: MVP의 마지막 Phase가 닫힌다. 미션/별 기준을 넓히려면 각 미션의
  `check`를 확장하거나 새 미션을 추가하고, 필요하면 이 ADR을 갱신한다.
