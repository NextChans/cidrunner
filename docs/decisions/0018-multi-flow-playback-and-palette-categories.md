# 0018. Multi-flow playback & categorized palette

- Status: Accepted
- Date: 2026-07-08
- Deciders: 차니, Claude

## Context

기존 재생(ADR 0012)은 진입점 하나(ALB 우선)의 단일 경로만 추적해, 3-tier와
서버리스가 공존하는 캔버스에서 나머지 경로는 보이지 않았다. "재생을 누르면
트래픽이 어떻게 흐르고 데이터가 어디 저장되는지" 전부 보여야 한다. 팔레트도
리소스 10종이 평평하게 나열되어 성격이 드러나지 않았다.

## Decision

1. **멀티 플로우 추적.** `simulate()`가 **모든 진입점**(각 ALB + 인바운드가 없는
   각 Lambda)에서 플로우를 하나씩 추적한다. 결과는 `flows[]`(경로·차단 지점·
   라벨)와 파생 집계(`edgeHops`, `arrivals`, `blockedNodeIds`)다. SG 엣지는
   트래픽이 아니므로 제외한다(ADR 0017).
2. **재생 시각화.**
   - 파티클: 경로 엣지마다 홉 순서(0.45s/홉)로 지연된 SVG `animateMotion`.
   - **도착 펄스**: 요청이 노드에 닿는 시점(`arrivals`)에 초록 링이 퍼진다 —
     "데이터가 저장되는" 순간이 보인다.
   - 배너에 **플로우 목록**: `1. Web ALB → App Server → Main DB — 도달 🟢` 식으로
     경로별 상태를 나열하고, 차단 플로우는 사유와 함께 빨간 노드로 표시한다.
3. **팔레트 카테고리.** `ResourceMeta.category`(네트워킹/컴퓨팅/데이터베이스/
   스토리지/보안)로 그룹핑해 AWS 콘솔의 구획을 따른다.
4. **미니맵 토글.** 작은 화면에서 캔버스를 가리는 미니맵을 버튼으로 켜고 끌 수
   있고, 모바일 뷰포트에서는 기본 꺼짐이다.

## Consequences

- **좋은 점**: 하나의 캔버스에서 웹 계층과 서버리스 트래픽이 동시에 재생되고,
  미션 판정(ADR 0014)이 `flows`로 정확해진다(경로별 검사).
- **나쁜 점 / 한계**: 플로우당 그리디 단일 경로는 유지된다 — 한 ALB가 여러
  대상에 연결돼도 첫 경로만 따라간다. 응답(리턴) 트래픽은 그리지 않는다.
- **후속 영향**: ADR 0012를 이 ADR이 확장한다. 분기 fan-out이나 응답 애니메이션이
  필요하면 `SimFlow`를 확장한다.
