# 0008. UI 문자열 한국어 하드코딩 (i18n 프레임워크 미도입)

- Status: Accepted
- Date: 2026-07-08
- Deciders: 차니, Claude

## Context

지금까지 게임 UI 문자열(버튼·라벨·툴팁·미션 설명·헤더 등)은 모두 영문으로
하드코딩되어 있었다. 그러나 초기 타깃 사용자는 한국 개발자 위주다. README는
영/한을 병기해 두었지만, 정작 손으로 만지는 UI가 영문이라 학습용 게임으로서의
접근성이 떨어졌다.

이 시점에서 고려한 힘(forces)은 다음과 같다:

1. **타깃이 한국 개발자다.** UI가 한국어여야 미션 설명·힌트가 바로 읽히고
   학습 효과가 높다.
2. **i18n 프레임워크(react-i18next, FormatJS 등) 도입은 오버엔지니어링이다.**
   현재 지원해야 할 언어는 사실상 하나(한국어)뿐인데, 메시지 카탈로그·로케일
   스위처·번들 증가를 떠안는 것은 YAGNI 위배다.
3. **AWS 리소스명·기술 용어는 원어로 남아야 한다.** `VPC`, `Subnet`, `EC2`,
   `Security Group` 등은 실제 AWS 콘솔·공식 문서에서 보는 이름과 일치해야
   학습이 전이된다. 이를 번역하면 오히려 학습 효과를 해친다.

## Decision

UI 문자열을 **한국어로 하드코딩**한다. 별도의 i18n 프레임워크는 도입하지 않는다.

경계는 다음과 같이 둔다:

- **한국어로 번역**: 버튼 라벨, 섹션 헤더, 툴팁, placeholder, 에러/안내 메시지,
  미션 title·description·goal·hint, 리소스 설명 문구(`description`).
- **원어 유지**: AWS 리소스명·기술 용어 — `VPC`, `Subnet`, `Internet Gateway`,
  `NAT Gateway`, `Security Group`, `Load Balancer`(ALB), `EC2`, `RDS`, `S3`,
  `Lambda`, `API Gateway`, `AZ`, `CIDR`, `L7` 등. 리소스 팔레트의 `label`,
  노드 카드에 표시되는 타입 문자열도 여기 포함된다.
- **README·docs·CHANGELOG는 이중언어 유지** — 이미 영/한 병기가 되어 있으므로
  이번 변경 대상이 아니다.
- 코드 주석·변수명·식별자는 개발자용이므로 손대지 않는다.

`index.html`은 `lang="ko"`로 바꾸고 `meta description`·`title`을 한국어로 갱신한다.

## Consequences

- **좋은 점**: 코드가 단순하다(문자열이 마크업에 직접 들어감). i18n 런타임·카탈로그가
  없어 번들 사이즈가 작고, 번역 문자열 추가/수정이 곧 코드 수정이라 개발 속도가 빠르다.
  AWS 용어를 원어로 남겨 콘솔·문서와의 학습 전이를 지켰다.
- **나쁜 점 / 유의점**: 영어권 사용자에게 UI 접근성이 낮다(README는 영문을 남기지만
  실제 UI는 한국어 전용). 문자열이 컴포넌트 곳곳에 흩어져 있어, 훗날 국제화가
  필요해지면 하드코딩 → i18n 리팩터 비용이 발생한다 — 초기 스코프상 이 비용을
  의도적으로 감수한다.
- **후속 영향**: GitHub 별 다수 확보·해외 이슈 유입 등 국제화 수요가 실제로
  관측되면 이 결정을 재검토하고, 그때 i18n 프레임워크 도입을 별도 ADR로 다룬다.
