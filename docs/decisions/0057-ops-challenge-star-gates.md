# 0057. 운영 챌린지 티어 — 비용·카오스를 별점 게이트로

- Status: Accepted
- Date: 2026-07-09
- Deciders: 차니, Claude

## Context

게임 디자인 전환(ADR 0051~0054)으로 비용·카오스·등급이라는 트레이드오프 축은
생겼지만 셋 다 **표시 전용**이었다 — [일일 회고](../meetings/2026-07-09-day-retro.md)의
열린 항목 1번이 "예산·카오스를 별점에 반영(회귀 주의)"이다. 기존 12개 미션의 별
게이트를 소급 변경하면 기록·fixture·기대치가 깨진다.

## Decision

**기존 미션은 불변, 새 축을 별점으로 소비하는 "운영 챌린지" 티어를 신설한다.**

- **⚡ 무중단 운영 (`ha-survival`)** — 카오스가 게이트다.
  ★1 `alb → (ec2|ecs|eks) → rds` 라이브 체인 · ★2 **모든 단일 AZ 장애에서 체인
  생존**(AZ ≥ 2; `applyAzFault` 후 재시뮬 + `liveChain` 재판정 — Multi-AZ
  페일오버·복제본 승격 경로 모두 인정) · ★3 +오류 0·월 $90 이내.
- **💸 린 서버리스 (`lean-serverless`)** — 비용이 게이트다.
  ★1 `apigw → lambda → (dynamodb|s3)` · ★2 **월 $5 이내**+오류 0 (ALB/NAT/EC2
  하나만 넣어도 터진다) · ★3 +보안 경고 0.
- **순차 게이트**: 기존 미션들의 가산식(独立 별)과 달리, 미션의 본질 축(생존/예산)을
  통과하지 못하면 상위 별을 딸 수 없다 — 테스트 작성 중 "단일 AZ인데 예산 별로
  2★"가 나오는 것을 보고 확정했다.
- 두 미션 모두 ADR 0030의 `steps`(라이브 체크리스트)를 탑재한다 — steps 사용
  미션이 tutorial 단독에서 3개로 늘며 관련 가드 테스트를 갱신했다.
- 판정은 전부 기존 순수 함수 재사용: `applyAzFault`/`graphAzs`(0052-53),
  `estimateMonthlyCost`(0051), `liveChain`(구조적 판정), `simulate(…, {deadNodeIds})`.

## Consequences

- **좋은 점**: "싸고 안 죽는 설계"가 처음으로 **채점된다** — 게임 루프 완성.
  기존 12미션 무회귀(게이트·fixture 불변). 카오스 패널은 리허설 도구라는 역할을
  얻는다.
- **나쁜 점 / 한계**: 순차 게이트는 이 티어만의 규칙이라 별점 의미가 미션마다
  다르다(카드 힌트로 완화). 생존 판정은 단일 AZ 장애만 본다(카오스 v1 한계 그대로).
  Well-Architected 등급(0054)은 아직 어느 별점에도 안 걸려 있다 — 다음 후보.
- **후속 영향**: 미션 14종. 등급을 소비하는 세 번째 운영 미션(예: "등급 A 달성")과
  강사용 커스텀 미션의 제네릭 별점 규칙에 이 순차 게이트 패턴을 재사용할 수 있다.
