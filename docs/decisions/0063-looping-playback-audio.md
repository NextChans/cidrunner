# 0063. 재생 사운드 루프 — 파티클과 함께 계속 재생

- Status: Accepted
- Date: 2026-07-10
- Deciders: 차니, Claude

## Context

ADR 0058은 재생 사운드를 **한 패스만** 재생하도록 정했다("파티클은 루프해도
소리가 무한 반복되면 소음(torture)"). 실제 사용 후 차니의 피드백: "소리가 처음에만
나는 게 아니라 계속 나면 좋을듯 해. 생동감 있게." 파티클은 `CYCLE = 1.4s`로 무한
반복되는데 소리만 한 번 나고 그쳐서, 재생을 오래 지켜볼 때 시청각이 어긋났다.

## Decision

**사운드 스케줄을 `CYCLE`(1.4s) 주기로 루프한다 — 반전(ADR 0058의 one-pass 무효화).**

- `soundPlan(sim)`(순수 계획)은 그대로. 플레이어만 바뀐다: 각 이벤트를
  `t0 + e.t + k·CYCLE`(k=0,1,2,…)로 반복 스케줄해, 이벤트가 **해당 노드의 반복되는
  도착 pulse(`sim-arrival 1.4s infinite`)와 위상 고정**되게 한다. 시각·청각이 같은
  1.4s 심장박동을 공유한다.
- **룩어헤드 스케줄러**: 항상 ~2 사이클을 미리 Web Audio 클록에 예약하고
  `setInterval(CYCLE)`로 한 사이클씩 채운다. Web Audio는 자체 클록으로 재생하므로
  타이머 지터가 소리를 흔들지 않는다.
- **누수 방지**: 오실레이터는 종료 시 `onended`로 `live` 배열에서 제거해 장시간
  루프에도 참조가 무한정 쌓이지 않게 한다.
- `stopSimSounds()`가 `loopTimer`를 clear + 예약분을 stop. 중지·음소거·시뮬 변경
  (카오스 재실행) 시 즉시 멈춘다. `CYCLE`은 TrafficEdge의 값과 반드시 일치(주석 명시).
- 마스터 게인 0.12 유지 — 연속 재생이라도 정보 전달용의 조용한 레벨.

## Consequences

- **좋은 점**: 재생 중 트래픽이 "살아 있는" 감각이 시각과 완전히 동기화된다. 성공/차단
  피드백이 지속적으로 들려 접근성에도 도움.
- **나쁜 점 / 한계**: ADR 0058이 우려한 "소음" 리스크를 감수한다 — 조용한 게인과
  즉시 음소거 토글(0058)로 완화. 아주 깊은 체인(도착 > CYCLE)은 사이클 경계에서
  이벤트가 겹치지만, 이는 파티클 스트림의 겹침과 동일해 오히려 정합적이다.
- **후속**: 필요 시 게인/톤 다양화, 또는 "한 번만/계속" 사용자 설정. 지금은 계속
  재생이 기본(차니 요청).
