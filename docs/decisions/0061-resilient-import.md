# 0061. 관대한 불러오기 — 모르는 리소스는 건너뛰고 알린다

- Status: Accepted
- Date: 2026-07-10
- Deciders: 차니, Claude

## Context

차니가 실제 prod topology JSON(PayNext, 36개 노드)을 업로드했는데 "불러오기 실패"가
났다. 원인: 그 파일에 cidrunner가 아직 모델링하지 않는 **`ecr`·`cloudtrail`** 노드가
있었고, `sanitizeSnapshot`은 모르는 타입을 하나라도 만나면 `return null`로 **전체
설계를 거부**했다(ADR 0020/0023의 엄격한 화이트리스트 재구축). 한 개의 미지원 블록이
34개의 정상 리소스를 통째로 못 쓰게 만든 것이다.

## Decision

**구조가 깨진 경우에만 하드 실패하고, 미지원 리소스 타입은 건너뛴 뒤 알린다.**

- **하드 실패(변경 없음)**: 최상위가 객체가 아님, `nodes`/`edges`가 배열이 아님,
  노드 id가 문자열이 아님, 알려진 타입인데 좌표가 숫자가 아님, id 중복 — 진짜 손상.
- **소프트 스킵(신규)**: `type`이 레지스트리에 없으면 그 노드를 버리고 `droppedIds`에
  기록. 버린 노드를 부모로 삼던 자식은 **캐스케이드**로 함께 버린다(고아 방지, 부동점).
  버린 노드에 닿는 엣지, 그리고 없는 노드를 가리키는 엣지(댕글링)도 이제 실패가 아니라
  스킵한다. `LoadedDesign.unsupportedTypes`로 버려진 타입을 반환해 인스펙터가
  "지원하지 않는 리소스는 제외했습니다: ecr, cloudtrail" 토스트를 띄운다.
- 보안 불변식 유지: 각 살아남은 노드/엣지는 여전히 화이트리스트로 재구축되므로,
  외부 JSON이 스토어에 임의 필드를 주입할 수 없다(ADR 0023). 관대해진 것은 "무엇을
  버리느냐"뿐, "무엇을 신뢰하느냐"가 아니다.

## Consequences

- **좋은 점**: 실제 인프라 export(도구가 무엇을 담든)를 최대한 불러온다 — 34/36 노드,
  17/19 엣지로 로드되고 무엇이 빠졌는지 명시된다. 침묵의 데이터 증발이 없다.
- **나쁜 점 / 한계**: 미지원 리소스와 그 연결은 캔버스에서 사라진다(토스트로 고지).
  미지원 컨테이너를 버리면 그 안의 전 계층이 캐스케이드로 사라질 수 있다(현재 케이스는
  리프라 무해).
- **후속**: `ecr`·`cloudtrail`(그리고 자주 요청되는 리소스)을 정식 리소스로 추가하면
  드롭 없이 완전히 로드된다 — apply-ready terraform emitter 작업이 수반된다.
