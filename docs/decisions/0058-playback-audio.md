# 0058. 재생 사운드 — 홉 틱·도착 차임·차단 버즈 (Web Audio 합성)

- Status: Accepted
- Date: 2026-07-09
- Deciders: 차니, Claude

## Context

재생(시뮬레이션)은 파티클·링·pulse 등 시각 피드백은 풍부하지만(ADR 0048·0049)
소리가 없어 "데이터가 흐른다"는 감각이 절반이다. 차니의 요청: "데이터가 흐를때
소리도 나면 좋을듯 한데..어때?"

제약 조건:

- **에셋 0**: 오디오 파일을 추가하면 번들·배포·라이선스 관리가 생긴다.
- **자동재생 정책**: 브라우저는 사용자 제스처 없이 `AudioContext` 시작을 막는다.
- **루프 소음**: 파티클 애니메이션은 무한 루프지만 소리는 무한 반복되면 소음이다.
- **교실 사용**: 강사/학생 시나리오에서 즉시 음소거가 가능해야 한다.

## Decision

**Web Audio API로 톤을 합성한다 — 오디오 파일 없이, 한 패스만, 음소거는 영속.**

- **순수 계획 + 얇은 플레이어 분리** ([simSounds.ts](../../src/audio/simSounds.ts)):
  `soundPlan(sim: SimResult): SoundEvent[]`은 순수 함수 — `sim.arrivals`
  (`HOP_SECONDS` 그리드)에서 이벤트 시각을 도출하므로 **파티클과 자동 동기화**된다.
  플레이어는 계획을 `AudioContext` 스케줄에 옮기기만 한다. 테스트는 계획만 검증
  (오디오 mock 불필요).
- **소리 3종, 전부 oscillator 합성**:
  - `tick` — 홉 도착마다 사인파 짧은 톤(70ms). **홉 순번마다 반음 두 개씩 상승**
    (440×1.1225^step)해 "데이터가 앞으로 나아간다"가 귀로 들린다.
  - `chime` — flow가 sink(저장)에 도달하면 2음 상승 차임(880→1108.7Hz) = 저장 성공.
  - `buzz` — 차단된 flow는 낮은 사각파 버즈(98Hz sawtooth) = 실패.
- **중복 제거**: fan-out으로 여러 flow가 같은 (시각, 홉)을 지나면 틱 1회만
  스케줄(키 `kind:t:step`). 이벤트 상한 64개(대형 그래프 소음 방지).
- **한 패스만**: 파티클은 루프해도 소리는 도착 시퀀스 1회만 재생하고 끝난다.
- **자동재생 합법**: `AudioContext`는 lazy 생성 — 항상 시작 버튼 클릭(사용자
  제스처) 콜스택 안에서 첫 생성되므로 정책에 걸리지 않는다.
- **연결점은 하나** ([SimAudio.tsx](../../src/components/SimAudio.tsx)):
  `simulation`+`soundOn`을 구독하는 null 렌더 컴포넌트. 시작 버튼이든 카오스 AZ
  토글 재시뮬이든 **시뮬이 생기는 모든 경로**를 한 곳에서 커버하고, 중지·음소거
  시 예약된 오실레이터를 즉시 stop한다.
- **음소거 영속**: `soundOn`을 persist partialize에 포함 — 교실에서 한 번 끄면
  새로고침해도 꺼진 상태 유지. 툴바 시작 버튼 옆 Volume2/VolumeX 토글
  (`aria-pressed`).

기각한 대안: 오디오 파일 샘플(에셋·라이선스 부담, 합성으로 충분), 루프 재생
(소음), Tone.js 등 라이브러리(수 KB 함수 두 개에 의존성 추가 불합리).

## Consequences

- **좋은 점**: 번들 증가 사실상 0(순수 TS ~150줄). 도착 그리드에서 시각을
  도출하므로 시뮬 로직이 바뀌어도(백트래킹·페일오버 등) 소리가 자동 추종.
  성공/실패가 귀로 구별돼 접근성에도 기여.
- **나쁜 점 / 한계**: 합성음이라 "게임 사운드" 수준의 풍부함은 없다. 볼륨
  조절은 on/off 토글뿐(마스터 게인 0.12 고정). 홉 수 63을 넘는 초대형 그래프는
  뒷부분 이벤트가 잘린다(상한).
- **후속 후보**: 승격/페일오버(0053) 전용 효과음, 볼륨 슬라이더, 배지 획득
  사운드.
