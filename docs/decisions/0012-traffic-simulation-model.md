# 0012. Traffic simulation model

- Status: Accepted
- Date: 2026-07-08
- Deciders: 차니, Claude

## Context

Phase 3은 "시작"을 누르면 요청이 토폴로지를 따라 흐르는 모습을 애니메이션하고,
경로가 끊긴 지점을 짚어주는 단계다. 실제 AWS 트래픽은 라우팅 테이블·보안그룹·
헬스체크 등 수많은 규칙에 좌우되지만, 이를 브라우저 게임에서 충실히 시뮬레이션하는
것은 범위를 크게 벗어난다. 어느 수준까지 모델링할지 정해야 한다.

## Decision

**연결(엣지) 기반의 그리디 단일 경로 추적**으로 모델을 좁힌다.
[`src/graph/simulate.ts`](../../src/graph/simulate.ts)의 `simulate(nodes, edges)`가:

1. **진입점(entry)** 을 고른다 — ALB를 우선, 없으면 인바운드 엣지가 없는 Lambda.
   진입점이 없으면 실패로 안내한다.
2. 진입점에서 엣지 방향을 따라 **처음 만나는 미방문 이웃**으로 한 홉씩 전진한다.
3. **싱크(sink)** — RDS 또는 S3 — 에 도달하면 성공(`client → LB → app → DB`).
4. 전진할 엣지가 없으면 그 노드를 **병목(blockedNodeId)** 으로 표시하고, 노드 타입에
   맞는 한국어 실패 메시지를 반환한다.

결과(`SimResult`)는 store의 `simulation`에 담기고, UI가 이를 구독한다.

- **애니메이션**: 모든 엣지는 커스텀 `TrafficEdge`로 렌더링되고, 경로에 속한 엣지는
  SVG `<animateMotion>` 파티클을 홉 순서대로 지연시켜 흐르듯 보여준다.
- **하이라이트**: 경로 노드는 초록, 병목 노드는 빨강(펄스)으로 표시한다.
- **배너**: 성공/실패 메시지를 캔버스 상단에 띄운다. Start 버튼은 실행 중 "중지"로
  토글된다.

## Consequences

- **좋은 점**: 사용자가 그린 그래프만으로 즉시 판정·시각화가 가능하고, Phase 1의
  엣지 규칙과 자연스럽게 이어진다. 순수 함수라 테스트가 쉽다. Phase 5 미션 클리어
  판정이 이 결과를 재사용한다.
- **나쁜 점 / 한계**: 그리디 단일 경로라 분기(한 ALB가 여러 대상에 연결)는 첫 경로만
  추적한다. 보안그룹 규칙·라우팅 테이블·IGW 유무·헬스체크는 **판정에 반영하지
  않는다** — "경로 단절"만이 실패 원인이다. 노드 설정 검증(Phase 2) 오류도 시뮬레이션
  성패에는 관여하지 않는다(미션 별점에서 별도로 반영).
- **후속 영향**: 더 정밀한 규칙(퍼블릭 경로에 IGW 필요, DB 접근에 SG 필요 등)이
  필요해지면 `simulate`를 확장하고 이 ADR을 갱신하거나 후속 ADR로 대체한다.
