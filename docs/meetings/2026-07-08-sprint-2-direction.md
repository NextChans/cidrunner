## 현장 발견 (김하영 시험 운영 보고)

- 🐛 **데이터 손실 버그**: 학생 공유 링크를 여는 순간 확인 없이 자동저장이 덮여
  수업 시연용 설계가 소실됨.
- **채점 마찰**: 공유 링크에 미션 ID·별점이 실리지 않아 매번 수동으로 미션을
  다시 선택하고 성적을 수기로 옮김.

- 김하영: **피드백 주석** — 학생 설계 노드에 첨삭 메모를 달아 URL로 반환.

## CEO 의사결정

**"에디터 기본기(Editor Fundamentals)" 스프린트를 즉시 실행한다.**

1. 🐛 **공유 안전화** (최우선 — 실사용 데이터 손실): 기존 작업이 있으면 확인 후
   대체, 같은 탭 붙여넣기(hashchange)도 처리. 공유 스냅샷에 **미션 컨텍스트**
   포함(채점 마찰 해소).
2. **undo/redo** — zundo, 제스처(드래그·리사이즈·타이핑) 당 1스텝이 되도록
   debounced-leading 히스토리 캡처(엔지니어 우려 반영). Ctrl+Z/Y + 툴바 버튼.
3. **첫 방문 온보딩** — SG 엣지 문법을 명시한 환영 오버레이 + 튜토리얼 미션 핸드오프.
4. **별점 최고기록** (B-lite) — 미션 카드에 "최고 ★n" 영구 표시.
5. **로드 경로 단일화** — 리하이드레이트를 sanitize 경유(단, 실패 시 로컬 데이터
   보존 — URL보다 로컬을 신뢰).

**이월/보류**: C는 다음 스프린트 1순위(김하영 최소 스펙 + 신규 미션부터 선언적
술어로 작성해 DSL 재료 축적 — 엔지니어 절충안 채택). 미션 채점 근거 피드백과
피드백 주석은 C와 함께 설계. **분석은 외부 서비스 선택이 필요해 오너(차니) 결정
대기.** D는 동결 유지(수요 pull 원칙).
