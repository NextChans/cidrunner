### QA-001 — detach/공유 로드로 만들어진 "부모 없는 불법 배치"가 어디에서도 검증되지 않음

- **도메인**: D2 (컨테인먼트/규칙), 파급: D3(미션 판정) · D4(Terraform)
- **재현**:
  1. 챌린지 모드에서 `글로벌 동적 웹`(global-web) 등 VPC를 requiredResources에 포함하지 않는 미션 선택.
  2. VPC 안에 ALB를 배치(정상). ALB 우클릭 → **"부모에서 분리"**.
  3. ALB가 최상위(캔버스)로 떨어짐 → 시작(시뮬) 실행 → R53→CF→ALB→EC2→RDS 도달 성공.
  4. 인스펙터·노드에 **오류 표시 없음** → 미션 ★2("설정 오류 없음") 판정됨.
  5. Terraform 내보내기.
- **예상**: `allowedParents`에 `'canvas'`가 없는 리소스(subnet/igw/nat/sg/alb/ec2/ecs/eks/rds/elasticache/efs)가 최상위에 있으면 **에러**로 표시되고 ★2·apply-ready가 막혀야 함.
- **실제** (probe로 확인):
  - `graphIssues([loose alb, loose ec2, loose subnet], [])` → **errors 0건** (경고만 발생).
  - loose ALB/EC2/subnet 각각 `graphErr=0`, per-node validate=0.
  - `simulate()`의 인터넷 인그레스 게이트는 VPC 없는 ALB를 **명시적으로 면제**([simulate.ts:99](../../src/graph/simulate.ts#L99) `if (!vpc) return null`)하므로 시뮬도 통과.
  - `generateTerraform()` 산출물이 `aws_vpc.REPLACE_ME.id`(ALB), `aws_subnet.REPLACE_ME.id`(EC2)를 참조 → `terraform validate` 실패 = **apply-ready 계약 위반**.
- **영향**:
  - 미션 ★2 "설정 오류 없음"이 구조적으로 불가능한(AWS에 존재할 수 없는 VPC-less ALB) 설계에서 통과 → 학습 목적 훼손.
  - "내보낸 HCL은 그대로 apply된다"는 핵심 약속(ADR 0016/0025/0044)이 조용히 깨짐 — REPLACE_ME는 원래 "일어나면 안 되는" 상태의 마커인데, 정상 UI 동작("부모에서 분리")으로 도달 가능.
- **추정 원인**: containment 합법성은 **생성 시점에만** 강제됨 — `addNode`([useGraphStore.ts:311](../../src/store/useGraphStore.ts#L311)), `onDrop`([Canvas.tsx:236](../../src/components/Canvas.tsx#L236))은 규칙을 확인하지만, `detachNode`([useGraphStore.ts:443](../../src/store/useGraphStore.ts#L443))는 합법성 가드가 전혀 없고, [checks.ts](../../src/graph/checks.ts)에도 "이 리소스는 X 안에 있어야 하는데 아니다"를 잡는 규칙이 없음. `normalizeContainment`는 **로드 시에만** 실행되고 이미 parent 있는 노드만 다룸(detach된 loose 노드가 컨테이너 밖으로 이동하면 재편입도 안 됨).
- **제안 픽스**: `graphIssues`에 containment-legality 규칙 1개 추가 — `!canBeTopLevel(t)`인데 `parentId`가 없거나, 부모 타입이 `allowedParents`에 없으면 error 푸시. (또는 `detachNode`에서 `canBeTopLevel` 아닐 때 분리 거부 + 노티스.) 규칙 레이어에 넣으면 detach·공유 URL·슬롯 로드·hand-edit 모두 한 번에 커버됨.
- **✅ Resolved by hotfix (`5fa3283`), [ADR 0045](../decisions/0045-containment-legality-validation.md)**: `graphIssues`에 상시 containment-legality 규칙 추가 — `!canBeTopLevel(t)`이고 해석 가능한 parent가 없으면(미설정 + dangling 모두) error. 미션 `allValid` 자동 게이팅, Terraform export error 차단 게이트, sim은 자연 정합. 신규 테스트 13개(`containment-legality.test.ts`), 회귀 0.

### QA-002 — greedy 단일 경로 트레이서가 분기 진입점에서 미션 클리어를 오탐

- **도메인**: D4(시뮬레이션) → D3(미션 판정)
- **재현** (probe로 확인):
  1. VPC/2 subnet/IGW/ALB 배치.
  2. ALB → EC2-A **(DB 미연결, 막다른 길)** 엣지를 **먼저** 그림.
  3. ALB → EC2-B → RDS 엣지를 나중에 그림 (유효한 완결 경로 존재).
  4. 시작(시뮬) 실행.
- **예상**: ALB→EC2→RDS 유효 경로가 존재하므로 `three-tier`/`container-workload` 미션이 도달로 판정(★1+).
- **실제**: `simulate()`가 진입점에서 **첫 번째 아웃고잉 엣지만 선택하고 백트래킹하지 않음** → path=`[alb, ec2-dead]`, `ok=false`. 미션 미클리어.
- **영향**: 미션 판정([MissionPanel.tsx:82](../../src/components/MissionPanel.tsx#L82))이 이 시뮬 결과를 그대로 사용. 플레이어가 HA로 앱 서버 2대를 두고 한쪽 DB 엣지를 빠뜨린 채 다른 경로는 완성한 경우, "올바른 경로가 있는데도" 미션이 안 열림. 시뮬 메시지가 막힌 EC2를 가리켜 힌트는 되지만, 판정은 false-negative.
- **추정 원인**: [simulate.ts:189-197](../../src/graph/simulate.ts#L189) — `for (const edge of trafficEdges) { ... break }` 첫 매치에서 확정, 대안 분기 미탐색. 진입점당 **1 flow, 1 path**. (ADR 0012/0018의 의도된 단순화이나, 분기가 있는 진입점의 미션 판정 영향은 문서에 없음.)
- **성격 참고**: 최소 정답(선형 체인)은 영향 없음 — 분기가 없으면 어떤 엣지를 골라도 sink 도달. 오탐은 "완결 경로 + 미완결 분기"가 공존하고 미완결 분기가 엣지 배열에서 앞설 때만 발생. 그럼에도 mission grading의 신뢰도 문제이므로 Major로 분류.
- **제안 픽스**: `traceFlow`를 DFS/백트래킹으로 바꿔 "sink에 도달하는 경로가 하나라도 있으면 ok"로 판정하거나, 최소한 미션 `check`에서 "path에 포함" 대신 "sink까지 도달 가능한 부분그래프 존재"를 별도 판정. F3 리팩터에서 시뮬 엔진 손대면 함께 처리 권장.

## Minor

### QA-003 — ALB ↔ 컨테이너(ECS/EKS)·Lambda 미연결이 산출물에서 "조용히" 누락

- **도메인**: D4(Terraform)
- **재현**: ALB→ECS→RDS(=`container-workload` 정답 토폴로지) 내보내기.
- **실제** (probe로 확인): ALB는 `aws_lb_target_group` + `aws_lb_listener`를 emit하지만 **타깃 그룹 어태치먼트가 0건**이고, ECS 서비스에도 `load_balancer` 블록이 없음 → ALB와 컨테이너가 HCL 상 **분리**됨.
- **원인/맥락**: 이는 **의도된 설계** — [ADR 0026](../decisions/0026-resource-expansion-2.md)이 "ECS/EKS는 self-contained, ALB 타깃 그룹 자동 등록 안 함(EC2=instance/Fargate=ip 타깃 타입 충돌)"으로 명시. `albTargets`가 `ec2`만 수집([terraform.ts:59-62](../../src/graph/terraform.ts#L59)).
- **왜 Minor로 남기나**: 문제는 "결정" 자체가 아니라 **일관성**임. 프로젝트의 원칙은 "불완전은 REPLACE_ME로 시끄럽게 표시"(ADR 0025/0044)인데, 이 갭은 산출물에 **아무 마커·주석도 없이** 정상처럼 보이는 빈 타깃 그룹을 남김. 사용자는 컨테이너를 직접 붙여야 한다는 안내를 HCL에서 못 받음. 또 `alb.connectsTo`에 `lambda`가 있으나([alb.ts:19](../../src/resources/alb.ts#L19)) ALB→Lambda 타깃도 동일하게 드롭되며 ADR 0026의 근거(타깃 타입 충돌)에도 언급 없음.
- **제안 픽스**: ALB emit 시 컨테이너/Lambda 타깃 엣지가 있으면 주석 `# NOTE: attach <ecs_svc>.load_balancer here — see ADR 0026` 또는 placeholder 어태치먼트 스텁을 함께 출력.

F3에서 핸들 늘릴 계획이면 스냅샷에 핸들 보존 권장.

### QA-008 — `bumpNodeSeq`는 `type-N` 패턴 id만 파싱 (외부 스냅샷 방어 부족)

- **도메인**: D5(영속)
